#
了解！原來你是要針對 Vite / Rollup / Webpack 打包配置 這一塊的專業術語。

這在「資深前端」的面試中非常關鍵，因為這代表你不只會寫 Code，還懂**「如何把 Code 交付給瀏覽器最有效率」**（Delivery Performance）。

以下是針對 打包與建置 (Build & Bundling) 相關的高級術語，請看看你是否在調整 Vite 設定檔 (vite.config.ts) 時做過這些事：

一、 減肥與除廢 (Size Optimization)
重點：讓打包出來的檔案越小越好。

1. Tree Shaking (搖樹優化)
這是什麼：想像把樹搖一搖，枯葉（沒用到的程式碼）就會掉下來。打包工具會自動分析 import，把你引用的套件中「沒用到的函式」剔除，不打包進去。

自我檢測：

你有沒有在用像 Lodash 或 UI Library 時，只 import { button } from 'lib' 而不是整包引入？

你是否確保你的 Code 沒有 Side Effects，讓 Vite 能夠放心地刪除死碼？

履歷寫法：

透過 Tree Shaking 機制剔除無用代碼，有效縮減 Bundle Size。

2. Minification / Uglification (代碼壓縮與混淆)
這是什麼：把變數名稱變短（const strictUserPermission -> const a），去掉空白和註解。Vite 預設是用 Esbuild 做這件事，以前 Webpack 是用 Terser。

自我檢測：

這通常是預設開啟的，但如果你有特別去調整 terserOptions 去掉 console.log 或 debugger，這就值得提。

履歷寫法：

配置 Build Process 自動移除 Console/Debugger 指令，並透過 Minification 優化生產環境代碼體積。

3. Gzip / Brotli Compression (壓縮傳輸)
這是什麼：打包完後，再用 Gzip 或 Brotli 演算法把它壓成 .gz 或 .br 檔，讓伺服器傳輸更快。

自我檢測：

你有裝過 vite-plugin-compression 這種插件嗎？

履歷寫法：

導入 Gzip/Brotli 壓縮策略，使靜態資源傳輸體積減少約 70%。

二、 分包策略 (Chunking Strategy)
重點：不要把所有雞蛋放在同一個籃子裡（不要全打包成一支超級大的 index.js）。

4. Code Splitting (代碼分割)
這是什麼：把程式碼切成好幾塊。最常見的是「路由懶加載」，使用者去哪一頁才載入那一頁的 JS。

自我檢測：

你的 Vue Router 裡面是用 import() 語法載入頁面的嗎？如果是，這就是 Code Splitting。

履歷寫法：

實作 Route-Level Code Splitting (路由級代碼分割)，大幅降低首屏載入時間 (FCP)。

5. Vendor Chunking / Split Chunks (第三方套件分包)
這是什麼：把 node_modules 裡面的東西（Vue, Axios, Lodash）獨立打包成一個 vendor.js。

為什麼要這樣做：因為你的業務邏輯天天改，但 Vue 版本不會天天升級。把 Vue 獨立出來，瀏覽器就可以把它快取 (Cache) 起來，使用者下次來就不用重新下載。

自我檢測：

你有在 vite.config.ts 裡的 build.rollupOptions.output.manualChunks 設定過東西嗎？或是把比較大的套件（如 Echarts）單獨拆出來？

履歷寫法：

優化 Chunk Split Strategy (分包策略)，將第三方依賴 (Dependencies) 與核心業務邏輯分離，最大化瀏覽器快取效益。

三、 監控與分析 (Analysis)
6. Bundle Analysis (打包體積分析)
這是什麼：你的專案越來越肥，到底是誰害的？

自我檢測：

你有用過 rollup-plugin-visualizer 或 webpack-bundle-analyzer 嗎？它會跑出一張很多格子的圖，告訴你哪個套件佔最大空間。

履歷寫法：

定期執行 Bundle Analysis 監控專案體積，識別並移除臃腫依賴 (Bloated Dependencies)。

💡 如何把這些「打包術語」加進你的履歷？
建議加在 「國立成功大學」 這一段，因為你提到你在這裡做了「新技術導入」與「Vite」。

修改前：

主導技術選型，將專案從舊制遷移至 Vite + Vue 3 + TypeScript，建立模組化開發標準。

修改後 (加入打包優化關鍵字)：

主導專案從舊制遷移至 Vite + Vue 3，重新設計 Build Pipeline。

實作 Code Splitting (代碼分割) 與 Vendor Chunking (分包策略)，並導入 Gzip 壓縮，成功將首屏載入體積縮減 40%，顯著提升 FCP 效能。

導入 Bundle Visualizer 建立體積監控機制，定期進行 Tree Shaking 與依賴優化，確保專案輕量化。

這樣寫的差異：
原本只說「換了 Vite」，現在變成「我知道怎麼調校 Vite 讓它效能最好」。這就是架構師與一般工程師的差別。
